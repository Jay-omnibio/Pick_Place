# Plot Guide: How To Read The 6 Output Plots

This guide explains each plot generated by `tools/plot_run_metrics.py`, what each line means, and what "good" behavior looks like.

## Common Axis Meaning
- `Step` (x-axis): control loop iteration index.
- `Meters` (y-axis for position/error plots): distance or position values.
- `Command` (y-axis for action plot): commanded movement/gripper values.

---

## 1) `01_reach_metrics.png` - Reach Metrics
Lines:
- `distance_to_object`: `||s_obj_rel||` (norm of object-relative belief vector).
- `reach_error`: `||s_obj_rel - [0, 0, -0.08]||` (error to pre-grasp relative target).

How to interpret:
- `reach_error` is the main one for current logic.
- Lower is better.

Good trend:
- `reach_error` decreases over time with limited oscillation.

Warning signs:
- Flat for long windows (stagnation).
- Big repeated spikes (recovery loops or unstable approach).

---

## 2) `02_object_relative_belief.png` - Object Relative Belief Components
Lines:
- `s_obj_rel_x`: believed object offset in x, relative to EE.
- `s_obj_rel_y`: believed object offset in y, relative to EE.
- `s_obj_rel_z`: believed object offset in z, relative to EE.

Target meaning:
- Current pre-grasp target is near `[0, 0, -0.08]`.
- So expected near end of Reach:
  - `s_obj_rel_x -> 0`
  - `s_obj_rel_y -> 0`
  - `s_obj_rel_z -> -0.08`

Good trend:
- `x,y` move toward 0.
- `z` moves toward `-0.08` (not necessarily 0).

---

## 3) `03_ee_belief.png` - EE Belief Components
Lines:
- `s_ee_x`, `s_ee_y`, `s_ee_z`: believed EE position in world frame.

How to interpret:
- This is belief-space EE trajectory, not direct ground truth.
- Use with plot 5 to compare belief vs true motion.

Good trend:
- Smooth, purposeful changes matching approach/lift phases.

Warning signs:
- Sharp sawtooth or repeated back-and-forth without improving reach metrics.

---

## 4) `04_actions.png` - Action Commands
Lines:
- `action_move_x`, `action_move_y`, `action_move_z`: commanded EE movement per step.
- `action_grip`: gripper command.
  - `1` = close
  - `0` = no-op
  - `-1` = open

How to interpret:
- Shows what policy/controller asks robot to do each step.
- Useful to diagnose oscillation or mode switching.

Good trend:
- Reach: mostly movement commands.
- Grasp: `action_grip` goes to `1`, movement becomes small/targeted.
- Lift: positive `action_move_z` with `action_grip=1`.

Warning signs:
- Repeated fast sign flips in `action_move_z` (may indicate recovery fighting constraints).

---

## 5) `05_true_positions.png` - True World Positions
Lines:
- Solid lines:
  - `true_ee_x`, `true_ee_y`, `true_ee_z` (actual EE world position)
- Dashed lines:
  - `true_obj_x`, `true_obj_y`, `true_obj_z` (actual object world position)

How to interpret:
- This is ground truth from simulator.
- Best plot for checking if robot actually moved where expected.

Good trend:
- EE approaches object position (with expected pre-grasp offset in z), then lift/place behavior appears.

Warning signs:
- EE nearly flat while commands are non-zero (kinematic/posture stall).

---

## 6) `06_phase_contact_gripper.png` - Phase, Contact, Gripper Observation
Lines:
- `phase`: task phase index (ytick labels show names like Reach/Grasp/Lift/Place).
- `obs_contact`: binary contact observation.
  - `1` means contact detected.
- `escape_active`: recovery state flag.
  - `0` = off
  - `1` = escape trigger
  - `2` = recovery active
- `obs_grip`: observed gripper width.

How to interpret:
- This is the timeline of state machine and contact events.

Good trend:
- Phase transitions happen in expected order.
- Contact rises near grasp attempts.
- `escape_active` appears occasionally, not continuously.

Warning signs:
- Stuck in Reach for long runs.
- Frequent Reach<->Grasp bouncing without lift.
- Continuous recovery activation.

---

## Quick Debug Recipe
1. Check `01_reach_metrics.png` for overall convergence.
2. Check `06_phase_contact_gripper.png` for phase/contact timeline.
3. If stuck, inspect `04_actions.png` for oscillation.
4. Confirm true motion in `05_true_positions.png`.
5. Use `02_object_relative_belief.png` to see which axis is blocking progress.
